## Open Source of the Week

This week’s focus is on a new R library - the **ggauto.** The ggauto is an open-source R library, by **[Nicola Rennie](https://www.linkedin.com/in/nicola-rennie?miniProfileUrn=urn%3Ali%3Afs_miniProfile%3AACoAABgaRBoBX0x49lDDMfBS4uojnebWWLH3QTk)**, that extends **ggplot2** by automatically selecting appropriate chart types and applying accessible default styling based on the structure of the data. Rather than requiring users to manually select and fine-tune visualizations, ggauto analyzes variable types (e.g., continuous, categorical, date) and generates a suitable plot with improved readability and default design. This helps streamline the visualization workflow and encourages better charting practices, particularly for common exploratory and reporting tasks.

Project repo: **[https://github.com/nrennie/ggauto](https://github.com/nrennie/ggauto)**

### **Key Features**

- Automatically selects **chart types** (e.g., bar, line, scatter, heatmap, distribution plots) based on the data structure

- Applies **accessible and opinionated styling defaults**, including improved color palettes, larger text, and better label formatting

- Supports multiple variables and adapts visualization types accordingly (e.g., faceting, grouped plots)

- Encourages a clear workflow by **separating data preprocessing from plotting**

- Returns standard **ggplot2 objects**, allowing further customization and layering

- Implements sensible defaults such as **ordering categories by magnitude** and symmetric axes when appropriate

More details are available in the project [documentation](https://nrennie.rbind.io/ggauto/).

License: MIT

## Book of the Week

This week’s focus is on an AI book - Transformers in Action by Nicole Koenigstein. This book, as the name implies, explains how transformer architectures and large language models (LLMs) work — and, more importantly, how to adapt, fine-tune, and deploy them in real applications. It balances theory, intuition, and executable code (via Jupyter notebooks), helping readers move from understanding attention mechanisms to building production-ready generative AI systems.

### **Topics covered**

- **Transformer fundamentals** — attention mechanisms, encoder–decoder architectures, and why transformers outperform earlier sequence models

- **Model families and architectures** — encoder-only, decoder-only, encoder-decoder, and emerging variants like mixture-of-experts

- **Text generation and prompting** — decoding strategies (beam search, sampling) and modern prompting techniques like zero-shot, few-shot, and chain-of-thought

- **Retrieval-Augmented Generation (RAG)** — integrating external knowledge sources into LLM workflows and understanding when to use RAG

- **Model alignment and training** — reinforcement learning from human feedback (RLHF), preference optimization, and evaluation strategies

- **Fine-tuning and efficiency** — parameter-efficient fine-tuning (LoRA, QLoRA), quantization, and optimizing models for performance and cost

- **Multimodal and specialized models** — working with models that handle text, images, audio, and video, plus smaller task-specific models

- **Scaling and optimization** — techniques for improving inference speed, memory usage, and long-context handling

- **Responsible AI practices** — bias detection, interpretability, and safeguarding LLMs in production systems

This book is ideal for data scientists, ML engineers, and technically inclined developers who already understand basic machine learning and want to deeply understand, customize, and deploy transformer-based models and LLM applications in practice.

The book is available for purchase on the **[publisher’s website](https://www.manning.com/books/transformers-in-action)** and on **[Amazon](https://www.amazon.com/Transformers-Action-Nicole-Koenigstein/dp/1633437884)**.
