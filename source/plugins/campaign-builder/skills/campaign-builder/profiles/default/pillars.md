# Content Pillars + 70/20/10

The skill uses pillars to (a) tag posts so the mix stays balanced and (b) help
choose template + framing when the source material spans multiple topics.

## Pillars and target mix

| Pillar | Target % | Examples |
| --- | --- | --- |
| **Production AI** | 35% | SQL agents, evaluation, monitoring, reliability, guardrails, AI pipelines that actually ship |
| **Docker for Data Scientists** | 25% | Dev Containers, Docker Compose, production readiness, image hygiene, multi-stage builds |
| **Learning Radar** | 25% | new courses, open-source projects, research papers, books — curation |
| **Forecasting & Data Science** | 15% | time series, feature engineering, model evaluation, forecasting pitfalls |

## 70/20/10 rule

| Type | Share | What it looks like |
| --- | --- | --- |
| **Teach (70%)** | the majority | original frameworks, lessons learned, engineering patterns, debugging stories — content only Rami can write |
| **Curate (20%)** | the next slice | sharing courses, papers, repos others made, with your take on why they matter |
| **Promote (10%)** | the smallest | your courses, books, newsletter |

## How to use this in a draft

When picking a template / framing for a post:

- **Production AI / Docker / Forecasting** → lean Teach. Use `lesson`,
  `contrarian`, `comparison`, or `list` templates. First-person experience is a
  strong signal.
- **Learning Radar** → lean Curate. Use `list` template. Add your take on what's
  novel — don't just announce the resource.
- **Promo** (course launch, book, newsletter) → use sparingly. Wrap in a Teach
  payload so the post still gives value to non-buyers.

## Quick check before saving

The "only Rami" filter (from `best-practices.md`):

> What unique insight can only Rami write?

If the answer is "anyone could post this", the post is in the Curate bucket —
fine in moderation, but flag it so the running mix stays under 20%.
