# Format: Threads post

**This is an ADAPTATION format.** The orchestrator has already drafted
the canonical LinkedIn version of this post. Your job: **condense the
LinkedIn body into a single Threads post** (or a 2-3 post thread when
the content needs it). Same voice, same hook, same content, same close.
Only the LENGTH changes.

Don't rewrite. Don't reframe. Same Rami voice as LinkedIn.

## Hard cap

**500 chars per post** — protocol-enforced.

## Adaptation strategy: condense first, split second

A 1,200-1,800 char LinkedIn post typically compresses to a single 500-char
Threads post — about a 3x reduction. Threads sits between LinkedIn (long)
and X (terse), so the compression is gentler than X's 280.

What stays from LinkedIn:

- The hook line.
- The 2-3 strongest facts / numbers / examples.
- The takeaway and (on CTA topics) the course link.

What gets cut: third-tier supporting examples, hedges, repeated framing,
long bulleted lists (compress to 2-3 representative items).

If the LinkedIn body has **multiple distinct beats** that cleanly split,
write a 2-3 post thread (each ≤ 500 chars). Single post is the default.

## Output structure

Single post:

```markdown
# Threads — <topic title>

{post text, ≤ 500 chars}
```

Thread (2-3 posts):

```markdown
# Threads thread — <topic title>

## 1/
{post text, ≤ 500 chars}

## 2/
{post text, ≤ 500 chars}

## 3/
{post text, ≤ 500 chars; course link here on CTA topics}
```

## Anti-patterns

- **Don't reframe for Threads.** Same Rami voice as LinkedIn.
- **Don't paste the LinkedIn version unchanged** — the 500 char cap
  forces compression. Don't truncate mid-sentence.
- **No engagement bait.** No "Repost if you agree", no "Follow for more".
- **No "🧵" / "thread incoming"** intros.
- **Hashtags:** skip them; Threads culture is light on tags.

## Validation

No structural validator. QA inline: ≤ 500 chars per post, voice matches
LinkedIn, no banned hype phrases.
