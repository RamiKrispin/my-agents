## Open Source of the Week

This week I’m focusing on **Gemma 4**, Google’s new open-weight LLM.

The Gemma 4 family supports a wide range of generative AI tasks, including question answering, summarization, and reasoning. While it’s not the first open-weight model we’ve seen (e.g., GPT-OSS, Qwen), it could mark a turning point for the open LLM ecosystem.

In my view, Gemma 4 signals a broader shift—one in which more companies may begin releasing open-weight models.

This family of models includes three main groups:

- **Small Sizes:** 2B and 4B effective parameter models built for ultra-mobile, edge, and browser deployment (e.g., Pixel, Chrome).

- **Dense:** A powerful 31B parameter dense model that bridges the gap between server-grade performance and local execution.

- **Mixture-of-Experts:** A highly efficient 26B MoE model designed for high-throughput, advanced reasoning.

Models on Hugging Face: **[https://huggingface.co/collections/google/gemma-4](https://huggingface.co/collections/google/gemma-4)**

### **Key Functionalities**

- **Reasoning:** All models in the family are designed as highly capable reasoners, with configurable thinking modes.

- **Extended Multimodalities:** Processes Text, Image with variable aspect ratio and resolution support (all models), Video, and Audio (featured natively on the E2B and E4B models).

- **Increased Context Window:** Small models feature a 128K context window, while the medium models support 256K.

- **Enhanced Coding & Agentic Capabilities:** Achieves notable improvements in coding benchmarks alongside built-in function-calling support, powering highly capable autonomous agents.

- **Native System Prompt Support:** Gemma 4 introduces built-in support for the system role, enabling more structured and controllable conversations.

## New Learning Resources

Here are some new learning resources that I came across this week.

### **MIT Introduction to Deep Learning 2026**

This is the 3rd introductory deep learning course MIT released recently. The MIT 6.S19 Introduction to Deep Learning course, taught by Prof. Alexander Amini and Prof. Ava Amini, runs at MIT every year and focuses on the foundations of deep learning. This course has been running since 2018, and it is nice to see how the course program evolves alongside the rapid development in the field of AI, particularly in GenAI. This full-semester course covers the following topics:

- Introduction to deep learning

- Deep sequence modeling (RNN, etc.)

- Transformers

- Computer vision

- Deep reinforcement learning

- LLM

- Fine-tune models

All the course materials are available online, and the video lectures are published weekly (the first two lectures are available).

### **Stanford CME296 Diffusion & Large Vision Models**

Stanford just released a new course focusing on diffusion and large vision models. The course is taught by Afshine Amidi and Shervine Amidi, and it covers the foundations of diffusion, score matching, and flow matching, modern architectures such as Diffusion Transformers, and methods for controllable image generation and evaluation. The course combines theory with practical insights into state-of-the-art generative models. Ideal for students with a background in linear algebra, probability, calculus, and machine learning.

## Book of the Week

This week’s focus is on a new DevOps book, The Cloud DevOps Engineer’s Guide by **[Thiago Maior](https://www.linkedin.com/in/thiago-maior?miniProfileUrn=urn%3Ali%3Afs_miniProfile%3AACoAABryzU8BiQO_CribJDfcPcEFfr86pJGs8vw)**. The book covers the full DevOps lifecycle, and it offers a hands-on guide to cloud DevOps, using tools such as Git, Docker, Terraform, Kubernetes, CI/CD, and AI-powered automation to build secure, scalable, production-ready systems.

### **Topics Covered**

- **DevOps and cloud fundamentals** — core concepts, DevOps culture, lifecycle, and cloud service models (IaaS, PaaS, SaaS)

- **Development and infrastructure setup** — working with Git, Bash, Docker, and cloud CLIs to create a production-ready environment

- **Infrastructure as Code (IaC)** — provisioning and managing infrastructure using Terraform and reusable modules

- **CI/CD pipelines** — building automated workflows with tools like GitHub Actions for testing and deployment

- **Container orchestration** — deploying and managing applications with Kubernetes and Helm

- **Security and DevSecOps** — implementing IAM, secrets management, encryption, and secure cloud configurations

- **Observability and reliability** — monitoring, logging, alerting, and incident response practices using SLOs/SLIs

- **Advanced DevOps practices** — GitOps, FinOps, AIOps, platform engineering, and cost optimization strategies

This book is ideal for aspiring and early-career DevOps engineers, cloud engineers, SREs, and software developers who want a hands-on, end-to-end guide to building and operating cloud-native systems using modern DevOps tools and practices.

The book is available for purchase on the **[publisher’s website](https://www.packtpub.com/en-pl/product/the-cloud-devops-engineers-guide-9781835461679)** and on **[Amazon](https://www.amazon.com/Cloud-DevOps-Engineers-Guide-cloud-native/dp/1835461670)**.
