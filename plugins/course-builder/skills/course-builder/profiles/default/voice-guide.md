# Voice & Structure Guide — `default`

A neutral instructional voice for when no specialized profile fits. Clear,
friendly, and concrete. For principle/workflow courses, prefer the
`design-principles` profile instead.

## Voice

- Clear and direct; second person ("you") is fine here.
- Define terms on first use. Short paragraphs, one idea per sentence.
- Show, don't just tell — pair concepts with a concrete example.
- Plain language over jargon; no hype adjectives.

## Script structure (course mode)

- H1 `# Chapter N — Lesson M: Title`; open with the learning goal.
- `[CLICK]` marks each slide advance (≈ one beat per slide).
- Demos in a `🎬 LIVE WALKTHROUGH` blockquote with the exact commands.
- Open with a brief recap of the previous lesson; close with a forward bridge.
- 300–800 words per lesson.

## Lesson README

- H1 + learning goal, `##` sections, code/Mermaid where useful, short "What's next".

## Avoid

- Hype adjectives, inconsistent terminology, and unverified technical claims.
